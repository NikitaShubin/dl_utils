# dl_utils: утилиты глубокого обучения / CV

## Структура репозитория
- Репозиторий — git-подмодуль, монтируется в `main_project/*/dl_utils`
- Пути в коде относительны расположения dl_utils; в Docker — `/workspace`

## Идеология
- является набором инструментов для ML/DL/CV-проектов,
- расширяется почти любой функциональностью, обладающей потенциалом к кросспроектному
  использованию

## Проверка кода и тестирование
- `./check-py.sh` — полный конвейер: ruff format → ruff check → mypy → pytest
  **Проверяет только файлы из белого списка** (`root_files` в начале скрипта).
  Чтобы добавить файл — обнови этот массив.
- `./check-infra.sh` — линтинг Docker/shell/Markdown
- основная конфигурация череков и линтеров содержится в pyproject.toml
- любые изменения в коде в конечном счёте должны проходить все проверки из ./check-*.sh

## Docker
- Рабочий контейнер с GPU + Jupyter + OpenCode; запуск через `docker/restart.sh`
- Внутри монтирует корень главного проекта в `/workspace`
- UID/GID подстраиваются автоматически через `cmd.sh`

## Ключевые модули
- `utils.py` — базовая зависимость для большинства модулей (файлы, img/video I/O)
- `labels.py` — конвертация меток/суперметок из Excel/CSV/TSV-шаблонов
- `pt_utils.py`, `tf_utils.py`, `keras_utils.py`, `onnx_utils.py` — утилиты DL-фреймворков
- `cvat.py`, `cvat_srv.py` — офлайн/онлайн работа с CVAT
- `samal.py`, `sam2al.py`, `gdinoal.py` — авторазметка (SAM/SAM2/GroundingDINO)

## Стиль кода
- **Знаки препинания в комментариях**: перед кодом → двоеточие в конце;
  в строке после кода → без знака; после комментируемого кода → точка.
- **Аннотации типов**: современный синтаксис (`X | Y`, встроенные дженерики).
  Избегать `from typing import ...`, если есть лаконичная современная альтернатива.
- **Лаконичность**: предпочитать современные синтаксические конструкции
  стандартной библиотеки Python многословным или процедурным альтернативам,
  если это не ухудшает читаемость. Если ту же логику можно выразить короче
  и яснее через оператор (`|`, `:=`), comprehensions, встроенные функции
  (`any`, `all`, `dict.get`/`setdefault`) или новые конструкции (`match/case`) —
  используй их.
